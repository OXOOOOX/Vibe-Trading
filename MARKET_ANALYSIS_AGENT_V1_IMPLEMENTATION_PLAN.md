# 市场分析方法与智能体 v1 实施追踪

> 状态：核心构筑与黄金回归已完成  
> 完成日期：2026-07-20  
> 适用范围：个股与 ETF 日报、周报的市场结构分析  
> 安全边界：只生成研究结论与监控候选；监控须人工启用；禁止自动交易

## 一、目标

把“最近若干交易日最高价/最低价”从主结论降级为可比较的基线。正式结论由确定性方法注册表生成候选，再由分析智能体选择方法、组织证据和执行反证审查；模型不得自行发明价格、日期、比例或交易动作。

## 二、已完成构筑

- [x] 建立统一方法注册表 `market-analysis-methods/1.0`。
- [x] 所有分析只使用截止日以前已经完成的日线，不读取未来数据。
- [x] 同时计算 5、20、60、120、250 个交易日的多周期结构。
- [x] 登记市场状态识别、多周期价格结构、已确认摆动点、波动归一化、触及反应证据五类方法。
- [x] 支撑和阻力采用区间表达，并登记证据评分、置信度、失效条件与原始引用。
- [x] 复权序列仅可用于分析，不得直接成为监控触发价。
- [x] ETF 强制检查相对强弱、跟踪误差、份额、折溢价和成分贡献；缺失时降低置信度并登记缺口。
- [x] 日报已有工作智能体接入同一方法快照与输出契约。
- [x] 周报在确定性门控通过后调用一次分析智能体，并要求加载 `market-analysis-method` 技能。
- [x] 分析智能体只能选择已登记的方法编号和点位候选编号。
- [x] 分析智能体必须同时输出支持证据、反对证据、失效条件和反证审查。
- [x] 智能体失败时降级为确定性报告，不中断正式报告生产。
- [x] Markdown/PDF 把机器枚举、内部编号和晦涩英文术语转换为中文；结构化 JSON 继续保留稳定机器值。
- [x] 关键点位、监控触发价和失效价使用同一份方法候选，禁止报告与监控口径分叉。
- [x] PDF 标题与后续表格保持在同一页，避免孤立标题。

## 三、方法与职责边界

| 层级 | 职责 | 是否允许生成数值 |
|---|---|---:|
| 确定性方法层 | 清洗日线、识别状态、计算结构、摆动点、波动区间和反应评分 | 是，仅来自冻结数据与登记公式 |
| 分析智能体 | 选择已登记方法和候选，解释跨周期冲突，组织正反证据 | 否 |
| 契约校验层 | 拒绝未知方法、未知点位、新增数字、缺失反证或非法交易语义 | 否 |
| 报告与监控层 | 将选中候选转成中文报告和人工确认的监控候选 | 否 |
